# HANDOFF作成ガイド

## 目的

各クライアントプロジェクトで作成するHANDOFF.mdの構造を標準化し、実装時の手戻りを防ぐ。

---

## 必須セクション

### 1. プロジェクト情報

| 項目 | 説明 |
|------|------|
| 企業名 | クライアント正式名称 |
| ディレクトリ | プロジェクトのパス |
| 作成日 | HANDOFF作成日 |
| テンプレート | 使用したテンプレート名 |

### 2. 企業基本情報

| 項目 | 説明 |
|------|------|
| 代表者 | 代表者名 |
| 設立 | 設立年 |
| 所在地 | 住所 |
| 電話番号 | 代表電話 |
| 事業内容 | 主要事業 |

### 3. ロゴ

| 項目 | 説明 |
|------|------|
| ファイル一覧 | 全ロゴファイルのパスと用途 |
| ヘッダー用 | 使用ファイル、サイズ（PC/SP） |
| フッター用 | 使用ファイル、サイズ |
| ファビコン用 | 使用ファイル |
| カラー | ロゴの色、反転が必要な場面 |

※ロゴが未定の場合は「未定」と明記し、決定次第更新する

### 4. ブランドカラー

| 項目 | 説明 |
|------|------|
| メインカラー | コード、用途 |
| アクセントカラー | コード、用途 |
| サブカラー | コード、用途 |
| テキストカラー | 濃淡それぞれのコード |

※実装時に視認性の問題が発生した場合は理由とともに更新する

### 5. ヘッダー仕様

| 項目 | 説明 |
|------|------|
| PC | ロゴ位置、ナビ構成、スクロール時の挙動 |
| SP | ロゴ位置、ハンバーガーメニューの仕様 |
| 背景 | 透明/白/色付きの切り替え条件 |

### 6. フッター仕様

| 項目 | 説明 |
|------|------|
| レイアウト | カラム数、各カラムの内容 |
| PC | 横並びの配置 |
| SP | 縦並びの配置 |
| 背景色 | 色コード |

### 7. ページ別セクション構成

各ページについて以下を記載：

| 項目 | 説明 |
|------|------|
| セクション順 | 上から順にセクション名を列挙 |
| 各セクション背景色 | 白/グレー/色付き |
| CTA配置 | CTAセクションの位置と背景色 |

### 8. 技術スタック注意事項

使用フレームワーク/ライブラリ固有の注意点を記載：

- 設定ファイルの場所
- カスタムクラスの定義方法
- 標準クラスとの競合
- ビルド時の注意点

### 9. 素材ファイル

| 項目 | 説明 |
|------|------|
| 画像ディレクトリ | パス |
| 画像一覧 | ファイル名と用途 |
| 未撮影/未生成 | 必要だが未準備の素材リスト |

### 10. 実装状況

| ページ | 状態 | 備考 |
|--------|------|------|
| 各ページ | 未着手/進行中/完了 | 特記事項 |

---

## 推奨セクション

必須ではないが、あると実装がスムーズになる項目：

- ターゲットユーザー
- コンバージョン目標
- SEOキーワード
- 競合サイト参考
- デザイン禁止事項（AIデザイン排除ルール等）

---

## 更新ルール

1. 実装中に仕様変更が発生したら即座にHANDOFFを更新する
2. 変更理由を簡潔に記載する（例：「視認性の問題で変更」）
3. 古い情報は削除するか取り消し線で残す

---

## チェックリスト

HANDOFF作成時に確認：

- [ ] ロゴファイルが全て記載されているか
- [ ] 各ロゴの用途（ヘッダー/フッター/ファビコン/OGP）が明記されているか
- [ ] ブランドカラーが全て定義されているか
- [ ] ヘッダー/フッターのレイアウトが具体的に記載されているか
- [ ] 各ページのセクション背景色が定義されているか
- [ ] CTAセクションの背景色が明記されているか
- [ ] 技術スタック固有の注意点が記載されているか
